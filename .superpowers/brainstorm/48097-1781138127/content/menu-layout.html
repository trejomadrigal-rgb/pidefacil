Es la primera pantalla que ve el cliente al escanear el QR. ¿Cómo se organizan las categorías y productos?

A

Pills de categoría + scroll continuo (Recomendado)

Barra de pills horizontales pegada arriba. Al scrollear el menú, las pills marcan automáticamente la categoría activa. Toca una pill para saltar a esa sección. Todos los productos visibles en una sola página.

Ventajas

  • Patrón familiar (Rappi, Uber Eats, DiDi Food)
  • El cliente ve todo el menú de un scroll
  • Fácil de navegar por sección

Desventajas

  • Página larga si hay muchas categorías

B

Tabs de categoría (una a la vez)

Tabs en la parte superior. Cada tab muestra solo los productos de esa categoría en un grid de 2 columnas. Más ordenado visualmente, pero el cliente debe tocar cada tab para explorar.

Ventajas

  • Más limpio visualmente
  • Grid 2 col aprovecha mejor el espacio

Desventajas

  • El cliente no ve el menú completo de un vistazo
  • Más clics para explorar todo

C

Acordeón de categorías

Cada categoría es una sección colapsable. El cliente expande las que le interesan. Útil para menús muy largos (10+ categorías).

Ventajas

  • Ideal para menús extensos
  • Muy ordenado

Desventajas

  • Más clics para ver productos
  • Menos visual, más "lista"
  • No es el estándar en apps de comida
